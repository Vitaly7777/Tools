# Настройка виртуального окружения Python

## Быстрый старт

Чтобы установить все необходимые библиотеки, выполните скрипт в зависимости от вашей операционной системы.

### Для Linux / macOS

Откройте терминал в папке проекта и выполните команду:

```bash
bash setup.sh
```

### Для Windows

Откройте командную строку (cmd) или PowerShell в папке проекта и выполните команду:

```bash
setup.bat
```

Скрипт сам создаст виртуальное окружение .venv, обновит pip и установит все библиотеки из файла requirements.txt.

---

## Работа с виртуальным окружением

После выполнения скрипта окружение уже активировано. В следующий раз вам нужно будет активировать его вручную.

### Инструкция для Linux / macOS

**Активация:**
Выполните команду:

```bash
source .venv/bin/activate
```

После этого в начале строки терминала появится приглашение (.venv).

**Деактивация:**
Чтобы выйти из виртуального окружения, выполните:

```bash
deactivate
```

**Удаление окружения:**
Если нужно полностью удалить .venv, выполните:

```bash
rm -rf .venv
```

### Инструкция для Windows

**Активация:**
Выполните команду:

```cmd
.venv\Scripts\activate
```

После этого в начале строки терминала появится приглашение (.venv).

**Деактивация:**
Чтобы выйти из виртуального окружения, выполните:

```cmd
deactivate
```

**Удаление окружения:**
Если нужно полностью удалить .venv, выполните:

```cmd
rmdir /s .venv
```

## Управление библиотеками

**Установить новую библиотеку:**

```
pip install название-библиотеки
```

**Сохранить все текущие библиотеки в файл requirements.txt:**

```
pip freeze > requirements.txt
```

## О кэшировании pip

Pip автоматически сохраняет скачанные пакеты в локальную папку (кэш), чтобы при повторной установке не скачивать их заново. Это ускоряет работу и экономит трафик.

По умолчанию кэш находится:
*   **Linux:** в папке ~/.cache/pip/
*   **Windows:** в папке %LocalAppData%\pip\cache

Обычно отключать кэш не нужно. Если всё же требуется (например, чтобы гарантированно скачать свежую версию), используйте ключ --no-cache-dir:

```bash
pip install --no-cache-dir -r requirements.txt
```

## Возможные ошибки и их решение

**Ошибка:** «python3 не установлен» (Linux/macOS) или «python не установлен» (Windows)
**Решение:** Скачайте и установите Python 3 с официального сайта.

**Ошибка:** «файл requirements.txt не найден»
**Решение:** Убедитесь, что вы запускаете скрипт из той папки, где лежит файл requirements.txt.

**Ошибка:** «pip не обновляется»
**Решение:** Попробуйте запустить терминал или командную строку с правами администратора.
