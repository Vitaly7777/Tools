**Техническое задание на разработку приложения «Fast Excel Viewer»**

---

### 1. Общее описание

Приложение для быстрого просмотра файлов Excel (`.xlsx`, `.xlsm`, `.xltx`, `.xltm`) с графическим интерфейсом на PySide6. Основная цель — максимально быстрый доступ к данным с фоновой подгрузкой и минимальной блокировкой интерфейса.

---

### 2. Архитектура и технологии

- **Язык:** Python 3.9+
- **GUI:** PySide6 (Qt6)
- **Чтение Excel:** openpyxl (режим `read_only=True`)
- **Конфигурация:** JSON-файл
- **Логирование:** модуль `logging`, вывод в файл и/или консоль
- **Потоки:** `QThread`, флаг остановки `threading.Event`

---

### 3. Конфигурация (файл `config.json`)

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `PREVIEW_ROWS` | int | 100 | Количество строк первичного блока, читается всегда для каждого листа |
| `LOAD_LIMIT` | int | 0 | Глубина вторичной загрузки: `0` — полная, `-1` — не грузить, `N > 0` — ограничение строк (включая `PREVIEW_ROWS`) |
| `STABLE_THRESHOLD` | int | 10 | Количество строк без изменения `filled_rows`, после которого значение считается предварительным |
| `CACHE_MEMORY_LIMIT_MB` | int | 2048 | Ограничение памяти под кэш данных (МБ) |
| `SPARSITY_WARNING_RATIO` | float | 0.5 | Порог отношения `filled_rows / max_row` для предупреждения о разреженности |
| `LOG_LEVEL` | str | "INFO" | Уровень логирования: `DEBUG`, `INFO`, `WARNING`, `ERROR` |
| `LOG_FILE` | str | "app.log" | Путь к файлу лога |

---

### 4. Интерфейс пользователя

#### 4.1. Главное окно

- **Заголовок:** `Fast Excel Viewer - [имя файла]`
- **Меню:**
  - Файл
    - Открыть... (`Ctrl+O`)
    - Перечитать (`Ctrl+R`)
    - Выход (`Ctrl+Q`)
  - Вид
    - Сбросить геометрию окна к значениям по умолчанию
- **Панель вкладок (`QTabWidget`):** по одной вкладке на каждый лист книги.
- **Строка состояния (`QStatusBar`):**
  - Формат: `имя_файла | размер | дата_изменения | Листов: N | Всего строк: M | Активный: имя_листа`
  - Размер файла: в МБ с одним десятичным знаком.
  - Дата изменения: локальный формат `ГГГГ-ММ-ДД ЧЧ:ММ`.

#### 4.2. Содержимое вкладки листа

Каждая вкладка содержит:

1. **Панель метрик листа (верхняя часть):**
   - `Всего строк (max_row):` значение
   - `Столбцов:` значение
   - `Эффективных строк:` значение + цветовая индикация состояния
   - `Заполненных строк:` значение + цветовая индикация состояния
   - `Прогресс загрузки:` текст `"Загружено X из Y (Z%)"`

2. **Таблица данных (`QTableView` + модель `QAbstractTableModel`):**
   - Вертикальный заголовок: номера строк (`1, 2, 3...`)
   - Горизонтальный заголовок: буквы столбцов (`A, B, C... ZZ`)
   - Поддержка виртуальной прокрутки и ленивой подгрузки (`canFetchMore`/`fetchMore`)

3. **Индикатор загрузки на вкладке:**
   - Спиннер или иконка `⏳` в заголовке вкладки при активной загрузке листа.

#### 4.3. Цветовая индикация метрик

**Для `Эффективных строк` и `Заполненных строк`:**

| Состояние | Цвет | Условие |
|-----------|------|---------|
| Загружается | Жёлтый | Предварительное значение (получено после `STABLE_THRESHOLD` строк без изменений) |
| Точно | Зелёный | Полная загрузка листа завершена |
| Не загружалось | Серый | Данные ещё не читались (только метаданные) |

**Индикация разреженности листа (фон метрики `Всего строк`):**

| Цвет | Условие (`ratio = filled_rows / max_row`) |
|------|---------------------------------------------|
| Зелёный | `ratio >= 0.9` |
| Жёлтый | `0.5 <= ratio < 0.9` |
| Оранжевый | `0.2 <= ratio < 0.5` |
| Красный | `ratio < 0.2` |

### 5. Модель данных листа (`SheetDataModel`)

Наследует `QAbstractTableModel`.

**Хранилище:**
- `_data: List[Tuple]` — кэш прочитанных строк. Каждая строка — кортеж значений ячеек.
- `_row_count: int` — общее количество строк, известное модели (увеличивается по мере подгрузки).

**Методы:**
- `rowCount()`, `columnCount()`, `data()`, `headerData()`
- `canFetchMore(index)`, `fetchMore(index)` — запускают сигнал на дозагрузку данных.

**Сигналы:**
- `needMoreRows(row_from: int, count: int)` — запрос к контроллеру на чтение следующей порции.

### 6. Управление данными и потоками

#### 6.1. Состояния листа

| Состояние | Описание |
|-----------|----------|
| `NOT_STARTED` | Метаданные прочитаны, данные не читались |
| `META_LOADED` | Метаданные загружены |
| `PREVIEW_LOADING` | Чтение первичного блока (`PREVIEW_ROWS`) |
| `PREVIEW_LOADED` | Первичный блок прочитан |
| `FULL_LOADING` | Чтение вторичного блока (остатка) |
| `FULL_LOADED` | Лист прочитан полностью |
| `PAUSED` | Загрузка приостановлена (пользователь переключил вкладку) |
| `ERROR` | Ошибка при чтении листа |

#### 6.2. Структуры данных на уровне приложения

**`SheetInfo` (для каждого листа):**
```python
@dataclass
class SheetInfo:
    name: str
    max_row: int
    max_col: int
    is_active: bool
    effective_rows: int = 0
    filled_rows: int = 0
    loaded_rows: int = 0           # сколько строк прочитано в кэш
    state: SheetState = SheetState.NOT_STARTED
    data_cache: List[Tuple] = field(default_factory=list)
    last_stable_row: int = 0       # для отслеживания STABLE_THRESHOLD
    stable_counter: int = 0
```

#### 6.3. Очередь задач (`TaskQueue`)

Управляет порядком загрузки листов.

**Типы задач:**
- `LOAD_PREVIEW` — загрузить первичный блок листа.
- `LOAD_FULL` — загрузить остаток листа (до `LOAD_LIMIT` или до конца).
- `CALCULATE_METRICS` — пересчёт эффективных/заполненных строк.

**Приоритеты (при бездействии пользователя):**
1. `LOAD_PREVIEW` активного листа.
2. `LOAD_PREVIEW` всех неактивных листов (в порядке индексов).
3. `LOAD_FULL` активного листа.
4. `LOAD_FULL` неактивных листов (в порядке индексов).

**При переключении вкладки:**
1. Текущая задача прерывается (выставляется `stop_event`).
2. Состояние текущего листа → `PAUSED`.
3. Задача для выбранного листа помещается в начало очереди (приоритет — текущий).
4. Прерванная задача возвращается в конец очереди своего типа.

#### 6.4. Воркер загрузки (`LoaderWorker`)

Единственный экземпляр `QThread`, обрабатывает задачи из очереди.

**Алгоритм работы:**
```
while очередь не пуста и не stop_event.is_set():
    задача = очередь.взять_следующую()
    if задача.тип == LOAD_PREVIEW:
        прочитать_первые_N_строк(лист, PREVIEW_ROWS)
    elif задача.тип == LOAD_FULL:
        продолжить_чтение(лист, LOAD_LIMIT)
    
    если stop_event.is_set():
        сохранить_прогресс()
        вернуть_задачу_в_очередь()
        break
```

**Методы чтения:**
- Используют `openpyxl` с `read_only=True`.
- `iter_rows(min_row=start_row, max_row=end_row, values_only=True)`
- После каждой прочитанной строки проверяют `stop_event`.
- Отправляют сигналы в UI: `rows_loaded(sheet_name, rows_data, start_index)`, `metrics_updated(sheet_name, effective, filled, is_final)`.

### 7. Алгоритм определения предварительного `filled_rows`

```
effective = 0
filled = 0
stable_counter = 0
last_filled = 0

для i, row в enumerate(iter_rows(), start=1):
    if any(cell not in (None, "") for cell in row):
        effective += 1
        filled = i
        if filled == last_filled:
            stable_counter += 1
        else:
            last_filled = filled
            stable_counter = 0
    
    если stable_counter >= STABLE_THRESHOLD и состояние не FULL_LOADED:
        отправить сигнал metrics_updated(..., is_final=False)  # жёлтый
```

### 8. Управление памятью кэша

**Ограничение:** `CACHE_MEMORY_LIMIT_MB`.

**Оценка потребления памяти:**
- `sys.getsizeof(row_tuple)` для одной строки.
- `total_cache_size = sum(getsizeof(r) for r in cache) + getsizeof(cache)`.

**При превышении лимита:**
1. Логируется предупреждение.
2. В статус-бар выводится сообщение: `⚠️ Превышен лимит кэша. Невидимые данные сброшены.`
3. Для всех листов в состоянии `PAUSED` или неактивных:
   - Сохраняется только `effective_rows` и `filled_rows`.
   - Кэш `data_cache` очищается.
   - Состояние сбрасывается до `PREVIEW_LOADED` (если был `FULL_LOADING`).
4. Активный лист не сбрасывается.

### 9. Обработка ошибок

#### 9.1. Ошибка открытия файла

- Файл не найден / битый / защищён паролем / не Excel.
- Отображение модального окна с текстом ошибки.
- Область вкладок пустая или с сообщением: `Не удалось открыть файл: причина`.

#### 9.2. Ошибка чтения конкретного листа

- Вкладка создаётся.
- Вместо таблицы отображается текст ошибки (например, `"Лист повреждён или содержит неподдерживаемые элементы"`).
- В заголовке вкладки — иконка ⚠️.
- Ошибка логируется.

#### 9.3. Предупреждения openpyxl

- Фильтруются через `warnings.filterwarnings`.
- При необходимости пишутся в лог с уровнем `DEBUG`.

### 14. Последовательность запуска приложения

1. Чтение `config.json`.
2. Настройка логирования.
3. Создание `QApplication`.
4. Создание главного окна.
5. Восстановление геометрии из `settings.json`.
6. Обработка аргументов командной строки.
7. Если файл передан — открыть.

**При открытии файла:**
1. Открыть книгу `openpyxl(read_only=True)`.
2. Собрать метаданные всех листов (имя, `max_row`, `max_col`, `is_active`).
3. Создать вкладки для всех листов.
4. Активировать вкладку активного листа.
5. Запустить `LoaderWorker`.
6. Поместить в очередь задачи согласно приоритетам.
