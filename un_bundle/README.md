# 📦 Пара утилит для упаковки и распаковки проектов

## bundle.py — упаковщик проекта
## unbundle.py — распаковщик бандлов

---

## 🔍 Назначение

Утилиты предназначены для:

  * **Упаковки** Python проекта в один текстовый файл (бандл) со всей структурой и содержимым
  * **Распаковки** бандла обратно в файловую структуру

Полезно для:

  * Отправки проекта по email или в мессенджер (один файл вместо папки)
  * Создания снапшотов проекта
  * Переноса кода между системами
  * Архивирования с сохранением структуры

---

## 📥 Упаковка проекта (bundle.py)

### Базовое использование

`python bundle.py`

Создаст файл `project_bundle.txt` в текущей директории, содержащий все `.py` файлы проекта и (по умолчанию) `.md` файлы.

### Параметры командной строки

`--include-md`              Включить .md файлы в бандл (по умолчанию)

`--no-md`                   Исключить .md файлы из бандла

`--include`, `-i`           Паттерн включения (можно несколько раз)

`--exclude`, `-e`           Паттерн исключения (можно несколько раз)

`--with-requirements`       Включить requirements.txt (по умолчанию)

`--no-requirements`         Исключить requirements.txt

`-o`, `--output`            Имя выходного файла

`--max-size`                Максимальный размер файла в МБ

`-v`, `--verbose`           Подробный вывод

`-h`, `--help`              Показать справку

### Примеры использования bundle.py

**Базовая упаковка (все .py и .md файлы)**

`python bundle.py`

**Только Python файлы (без Markdown)**

`python bundle.py --no-md`

**Свой выходной файл**

`python bundle.py -o my_project.txt`

**Только файлы из папки src**

`python bundle.py --include "src/**/*.py"`

**Исключить тестовые файлы**

`python bundle.py --exclude "*test*" --exclude "**/tests/*"`

**Комбинация параметров**

`python bundle.py --include "app/**/*.py" --exclude "**/migrations/*" -o app_bundle.txt`

**Ограничить размер файлов**

`python bundle.py --max-size 5`

**Подробный вывод (показывает процесс)**

`python bundle.py -v`

### Конфигурационный файл

Можно создать файл `.bundle.config.json` в корне проекта:

```json
{
    "include_md": false,
    "include_patterns": ["src/**/*.py", "README.md"],
    "exclude_patterns": ["**/test_*.py", "**/migrations/*"],
    "output_file": "my_bundle.txt",
    "include_requirements": true,
    "max_file_size_mb": 10
}
```

**Приоритет параметров**

Для каждого параметра действует простое правило:

  * Если параметр указан в командной строке → берётся из командной строки
  * Если не указан в командной строке, но есть в конфиге → берётся из конфига
  * Если нет ни там, ни там → берётся значение по умолчанию

## 📤 Распаковка бандла (unbundle.py)

### Базовое использование

`python unbundle.py project_bundle.txt`

Распакует все файлы из бандла в текущую директорию, восстанавливая структуру папок.

### Параметры командной строки

`bundle`                    Путь к файлу бандла (по умолчанию: project_bundle.txt)

`-o`, `--output-dir`        Директория для распаковки

`-d`, `--dry-run`           Режим проверки (без записи)

`-v`, `--verbose`           Подробный вывод

`--overwrite`               Перезаписывать существующие файлы без подтверждения

`--list-files`              Только показать список файлов в бандле

`--no-skip`                 Не пропускать служебные секции

`-h`, `--help`              Показать справку

### Примеры использования unbundle.py

**Просто распаковать**

`python unbundle.py project_bundle.txt`

**Посмотреть, что внутри (без распаковки)**

`python unbundle.py project_bundle.txt --list-files`

**Проверить, что будет распаковано**

`python unbundle.py project_bundle.txt --dry-run`

**Распаковать в другую папку**

`python unbundle.py project_bundle.txt -o ../restored/`

**Распаковать с перезаписью всех файлов (без вопросов)**

`python unbundle.py project_bundle.txt --overwrite`

**Распаковать всё, включая служебные секции**

`python unbundle.py project_bundle.txt --no-skip`

**Подробный вывод процесса распаковки**

`python unbundle.py project_bundle.txt -v`

### Интерактивный режим

При распаковке, если файл уже существует, скрипт спросит:

`📝 Файл уже существует: src/main.py`
`   Перезаписать? (y/n/a): `

  * `y` - перезаписать этот файл
  * `n` - пропустить этот файл
  * `a` - перезаписать все (all)

## 🔄 Полный цикл работы

### Пример 1: Простой проект

# 1. Упаковываем проект
cd my_project
python bundle.py

# 2. Отправляем коллеге project_bundle.txt

# 3. Коллега распаковывает
python unbundle.py project_bundle.txt


### Пример 2: Выборочная упаковка

# Упаковываем только исходники, без тестов
python bundle.py --include "src/**/*.py" --exclude "**/tests/*" -o source_bundle.txt

# Смотрим, что внутри
python unbundle.py source_bundle.txt --list-files

# Распаковываем в отдельную папку
python unbundle.py source_bundle.txt -o ../source_only/


### Пример 3: Работа с конфигом

# Создаём конфиг .bundle.config.json
echo '{"include_md": false, "output_file": "code_only.txt"}' > .bundle.config.json

# Упаковываем (конфиг применится автоматически)
python bundle.py

# Временное переопределение
python bundle.py --include-md

## 📋 Формат бандла

Создаваемый файл имеет структуру:

```text
#================================================================
# BUNDLE: ИмяПроекта
# Создан: 2024-01-01 12:00:00
# Python файлов: 5
# Всего файлов: 7
#================================================================

КОНФИГУРАЦИЯ БАНДЛА
INCLUDE_MD: True
...

СТРУКТУРА ПРОЕКТА
📁 project/
├── 📁 src/
│   ├── 🐍 main.py
│   └── 🐍 utils.py
...

СТАТИСТИКА ПО ФАЙЛАМ
...

СОДЕРЖИМОЕ ФАЙЛОВ
