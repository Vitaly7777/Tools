# Мониторинг Файлов с `file_monitor.py`

`file_monitor.py` - это простой скрипт Python, предназначенный для отслеживания изменений в файлах в указанных каталогах. Он сканирует заданные пути, сохраняет текущее состояние файлов, сравнивает его с предыдущим состоянием (если такое имеется) и логирует все обнаруженные изменения.


## Начало работы
### Требования

Для запуска `file_monitor.py` не требуются дополнительные библиотеки.
Для `schedule_file_monitor.py` требуется библиотека `schedule`.

### Установка

1.  **Клонируйте репозиторий**  (скачайте файлы напрямую).
2.  **Создайте виртуальное окружение** (рекомендуется):
    python -m venv .venv
    **Активируйте виртуальное окружение**
    ***Для Linux/macOS***
    source .venv/bin/activate  

    или
    ***Для Windows***
    .venv\Scripts\activate  
  
3.  **Установите зависимости для `schedule_file_monitor.py`**:
    pip install schedule==1.2.2


### Структура Директорий

Для корректной работы предполагается следующая структура файлов:
.
├── .venv/                   # Виртуальное окружение
├── config.json              # Файл конфигурации
├── file_monitor.py          # Основной скрипт мониторинга
├── schedule_file_monitor.py # Скрипт для автоматизации запуска
├── logs/                    # Каталог для логов (создайте вручную!)
├── states/                  # Каталог для файлов состояния (создайте вручную!)
├── remove_files/            # Каталог для списка удаленных файлов (создайте вручную!)
└── (другие файлы)


**Важно:** Убедитесь, что каталоги `logs/`, `states/` и `remove_files/` существуют в той же директории, что и скрипты, прежде чем запускать монитор. В противном случае, скрипт попытается их использовать и может столкнуться с ошибками.

Путь к `config.json` задается в начале `file_monitor.py` в строке `CONFIG_JSON = 'config.json'`. Если ваша структура отличается от предложенной, вам может потребоваться скорректировать этот путь.

## Конфигурация

Все настройки монитора хранятся в файле `config.json`.

### Пример `config.json`

```json
{
  "monitor_directory": "'/mnt/dfs/РУЗ/Общие_вопросы_ЦК', '/mnt/dfs/РУЗ/Единая_методология_РУЗ'",
  "state_file": "./states/state.json",
  "log_file": "./logs/monitor.log",
  "removed_files_file": "./remove_files/removed_files.txt",
  "rotare_state_files": 10,
  "rotare_remove_files": 10,
  "rotare_log_files": 10,
  "ignore_dirs": "'.venv', 'venv', 'tmp', 'temp'",
  "ignore_patterns":  "'*.tmp', '*.temp', '~*', '*$~*', '.~*'"
}
```

### Описание полей `config.json`

*   **`monitor_directory`**: Строка, содержащая список путей к каталогам, которые будут сканироваться. Пути разделяются запятыми и заключаются в одинарные кавычки.
    *   Пример: `"'/path/to/dir1', '/path/to/dir2'"`
*   **`state_file`**: Путь к файлам, в которых хранятся текущие состояния файлов в отслеживаемых каталогах. После каждого успешного сканирования текущее состояние записывается в этот файл. Предыдущее состояние будет переименовано (см. `rotare_state_files`). Для каждого сканируемого каталога создается свой файл состояния.
    *   Пример: `"./states/state_dir1.json", "./states/state_dir2.json"`
*   **`log_file`**: Путь к файлу для записи логов выполнения скрипта. Логи дописываются при каждом запуске.
Для каждого сканируемого каталога создается свой файл логов.
    *   Пример: `"./logs/monitor_dir1.log", "./logs/monitor_dir2.log"`
*   **`removed_files_file`**: Путь к файлам для сохранения списка обнаруженных удаленных файлов.
Для каждого сканируемого каталога создается свой файл удаленных файлов.
    *   Пример: `"./remove_files/removed_files_dir1.txt", "./remove_files/removed_files_dir2.txt"`
*   **`rotare_state_files`**: Целое число, определяющее количество файлов состояний, которые будут храниться при ротации. Например, `10` означает, что будут сохранены `state.json`, `state_01.json`, ..., `state_10.json`.
*   **`rotare_remove_files`**: Целое число, определяющее количество файлов со списками удаленных файлов, которые будут храниться при ротации.
*   **`rotare_log_files`**: Целое число, определяющее количество файлов логов, которые будут храниться при ротации. При значении rotare_log_files равным нулю, логирование осуществляется дозаписью в файл лога.
*   **`ignore_dirs`**: Строка, содержащая список подкаталогов, которые будут исключены из сканирования. Каталоги разделяются запятыми и заключаются в одинарные кавычки.
    *   Пример: `".venv', 'venv', 'tmp', 'temp"`
*   **`ignore_patterns`**: Строка, содержащая список шаблонов файлов (например, расширений), которые будут игнорироваться при сканировании. Шаблоны разделяются запятыми и заключаются в одинарные кавычки.
    *   Пример: `*.tmp, *.temp, ~*` (игнорирует файлы с расширением .tmp, .temp и временные файлы, начинающиеся с `~`)

## Использование `file_monitor.py`

### Запуск вручную

Для ручного запуска `file_monitor.py` откройте терминал в каталоге, содержащем скрипт, и выполните:

.venv/bin/python file_monitor.py

или, если вы активировали виртуальное окружение:

python file_monitor.py

Скрипт выполнит однократное сканирование и зафиксирует изменения. Проверьте файлы логов (`monitor.log`)  и файлы со списками удаленных файлов (`removed_files.txt`) для просмотра результатов.

## Автоматизация с `schedule_file_monitor.py`

Для регулярного запуска `file_monitor.py` используйте `schedule_file_monitor.py`. Этот скрипт использует библиотеку `schedule` для выполнения задач по расписанию.

### Использование
1. Откройте терминал в директории с файлами `file_monitor.py`, `schedule_file_monitor.py` и каталогом `.venv`.(`  cd /path/to/dir  `)
2.  Убедитесь, что вы находитесь в директории с файлами `file_monitor.py`, `schedule_file_monitor.py` и каталогом `.venv`.  (`  ls -a `)
3.  Запустите скрипт из терминала:

    .venv/bin/python schedule_file_monitor.py


    или, если вы активировали виртуальное окружение:

    python schedule_file_monitor.py


4.  Вы можете свернуть окно терминала, но не закрывать его. Скрипт будет работать в фоновом режиме, запуская `file_monitor.py` согласно заданному расписанию.

5. Выход из задачи `schedule_file_monitor.py`(прерывание расписания запуска `file_monitor.py`) осуществляется комбинацией клавиш `Ctrl + C`.

### Настройка Расписания

Расписание задается в `schedule_file_monitor.py` строкой:

schedule.every().day.at("09:00").do(my_job)


Вы можете изменить эту строку для настройки расписания. Подробные опции для установки расписания доступны в [документации библиотеки `schedule`](https://schedule.readthedocs.io/en/stable/) или в файле `option_schedule.md` (есть в этом каталоге).

Примеры расписаний:

*   `schedule.every(10).minutes.do(my_job)`: каждый 10 минут
*   `schedule.every().hour.do(my_job)`: каждый час
*   `schedule.every().monday.do(my_job)`: каждый понедельник
*   `schedule.every().wednesday.at("13:15").do(my_job)`: каждую среду в 13:15

## Дополнительно

### Логирование

`file_monitor.py` записывает информацию о своей работе в файл, указанный в `log_file` (по умолчанию `./logs/monitor.log`). В логи будет включена информация о пропущенных временных файлах и о пропущенных каталогах.

### Ротация Файлов

*   **Файлы состояний**: Скрипт управляет ротацией файлов состояний. При каждом запуске, текущий `state.json` становится предыдущим (`state_01.json`), а старые файлы сдвигаются. Количество хранимых файлов определяется `rotare_state_files`.
*   **Списки удаленных файлов**: Аналогично, `removed_files_file` также подвергается ротации согласно `rotare_remove_files`.

### Игнорирование Каталогов/Файлов

Скрипт автоматически пропускает каталоги и файлы, указанные в `ignore_dirs` и `ignore_patterns` вашего `config.json`. Это полезно для исключения временных файлов, системных каталогов (`.venv`) или других ненужных элементов из мониторинга.
