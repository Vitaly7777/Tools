# Мониторинг Файлов с `file_monitor.py`

`file_monitor.py` - Этот Python-скрипт предназначен для мониторинга изменений в указанных директориях файловой системы. Он отслеживает добавленные, удаленные и измененные файлы, игнорируя определенные каталоги и шаблоны файлов. Результаты мониторинга, включая обнаруженные удаленные и измененные файлы, сохраняются в файлы, а процесс логируется. Реализована ротация логов и файлов состояний для управления дисковым пространством.

## Возможности ##

Мониторинг нескольких заданных директорий.
Отслеживание добавленных файлов.
Отслеживание удаленных файлов.
Отслеживание измененных файлов (по дате модификации).
Игнорирование специфических каталогов (регистронезависимое).
Игнорирование файлов по заданным шаблонам (например, временных файлов).
Сохранение состояния файловой системы в JSON-файлы для последующего сравнения.
Запись списка удаленных файлов в отдельные текстовые файлы.
Запись списка измененных файлов в отдельные текстовые файлы с указанием старой и новой дат модификации.
Ротация файлов логов, состояний, списков удаленных и измененных файлов для управления дисковым пространством.
Гибкая настройка через файл config.json.
Ведение подробных логов выполнения скрипта.

## Начало работы
### Требования

Для запуска `file_monitor.py` не требуются дополнительные библиотеки.
Для `schedule_file_monitor.py` требуется библиотека `schedule`.

### Установка

1.  **Клонируйте репозиторий**  (скачайте файлы напрямую).
2.  **Создайте виртуальное окружение** (рекомендуется):
    python -m venv .venv
    **Активируйте виртуальное окружение**
    ***Для Linux/macOS***
    source .venv/bin/activate  

    или
    ***Для Windows***
    .venv\Scripts\activate  
  
3.  **Установите зависимости для `schedule_file_monitor.py`**:
    pip install schedule==1.2.2


4. **Настройка конфигурации**
Перед запуском скрипта необходимо настроить файл `config.json`, который расположен в той же директории, что и скрипт `file_monitor.py`.

### Пример `config.json`

{
    "monitor_directory": "'/path/to/dir1', '/path/to/dir2'",
    "state_file": "./states/state.json",
    "log_file": "./logs/monitor_files.log",
    "removed_files_file": "./remove_files/removed_files.txt",
    "changed_files_file": "./change_files/changed_files.txt",
    "rotare_state_files": 10,
    "rotare_remove_files": 10,
    "rotare_change_files": 10,
    "rotare_log_files": 10,
    "ignore_dirs": "'.venv', 'venv', 'tmp', 'temp'",
    "ignore_patterns": "'*.tmp', '*.temp', '~*', '*$~*', '.~*'",
    "stream_on": 1
}

### Описание полей `config.json`

*  **`monitor_directory`**: (Обязательный) Строка, содержащая пути к директориям, которые необходимо отслеживать. Пути должны быть разделены запятыми и заключены в одинарные кавычки. 
    *   Пример: `"'/mnt/dfs/РУЗ/Общие_вопросы_ЦК', '/mnt/dfs/РУЗ/Единая_методология_РУЗ'"`.
*  **`state_file`**: (Обязательный) Путь к базовому файлу для сохранения текущего состояния файловой системы. 
    *   Для каждой отслеживаемой директории будет создаваться свой файл состояния с суффиксом, основанным на названии директории (например, `state_Общие_вопросы_ЦК.json`).
*  **`log_file`**: (Обязательный) Путь к файлу для записи логов скрипта.
*  **`removed_files_file`**: (Необязательный) Путь к базовому файлу для записи списка удаленных файлов. Аналогично `state_file`, для каждой директории будет свой файл (например, `removed_files_Общие_вопросы_ЦК.txt`). 
    *   Если не указан или пуст, список удаленных файлов не сохраняется.
*  **`changed_files_file`**: (Необязательный) Путь к базовому файлу для записи списка измененных файлов. Для каждой директории будет свой файл (например, `changed_files_Общие_вопросы_ЦК.txt`). 
    *   Если не указан или пуст, список измененных файлов не сохраняется.
*  **`rotare_state_files`**: (Необязательный, по умолчанию 0) Количество версий файлов состояний, которые будут храниться (включая текущую ротированную). 
    *   Например, 10 означает, что, помимо только что ротированного `state_01.json`, будет храниться 9 предыдущих ротированных версий (с суффиксами _02 до _10). Если 0, ротация не выполняется.
*  **`rotare_remove_files`**: (Необязательный, по умолчанию 0) Аналогично для файлов списка удаленных файлов.
*  **`rotare_change_files`**: (Необязательный, по умолчанию 0) Аналогично для файлов списка измененных файлов.
*  **`rotare_log_files`**: (Необязательный, по умолчанию 0) Аналогично для файлов логов.
*  **`ignore_dirs`**: (Необязательный) Строка, содержащая список названий каталогов, которые следует игнорировать при сканировании (регистронезависимо), разделенные запятыми и заключенные в одинарные кавычки.
*  **`ignore_patterns`**: (Необязательный) Строка, содержащая список шаблонов файлов (используется `fnmatch`), которые следует игнорировать, разделенные запятыми и заключенные в одинарные кавычки. 
    *   Например, `'*.tmp'` будет игнорировать все файлы с расширением .tmp.
*  **`stream_on`**: (Необязательный, по умолчанию 1) 1 для вывода логов в консоль во время выполнения, 0 для отключения вывода в консоль (логи будут только в файле).

## Использование `file_monitor.py`

### Запуск вручную

Для ручного запуска `file_monitor.py` откройте терминал в каталоге, содержащем скрипт, и выполните:

.venv/bin/python file_monitor.py

или, если вы активировали виртуальное окружение:

python file_monitor.py

Скрипт выполнит однократное сканирование и зафиксирует изменения. Проверьте файлы логов (`monitor.log`), файлы со списками удаленных файлов (`removed_files.txt`) и файлы со списками измененых файлов (`changed_files.txt`) 
для просмотра результатов.

## Автоматизация с `schedule_file_monitor.py`

Для регулярного запуска `file_monitor.py` используйте `schedule_file_monitor.py`. Этот скрипт использует библиотеку `schedule` для выполнения задач по расписанию.

### Использование
1. Откройте терминал в директории с файлами `file_monitor.py`, `schedule_file_monitor.py` и каталогом `.venv`.(`  cd /path/to/dir  `)
2.  Убедитесь, что вы находитесь в директории с файлами `file_monitor.py`, `schedule_file_monitor.py` и каталогом `.venv`.  (`  ls -a `)
3.  Запустите скрипт из терминала:

    .venv/bin/python schedule_file_monitor.py


    или, если вы активировали виртуальное окружение:

    python schedule_file_monitor.py


4.  Вы можете свернуть окно терминала, но не закрывать его. Скрипт будет работать в фоновом режиме, запуская `file_monitor.py` согласно заданному расписанию.

5. Выход из задачи `schedule_file_monitor.py`(прерывание расписания запуска `file_monitor.py`) осуществляется комбинацией клавиш `Ctrl + C`.

### Настройка Расписания

Расписание задается в `schedule_file_monitor.py` строкой:

schedule.every().day.at("09:00").do(my_job)


Вы можете изменить эту строку для настройки расписания. Подробные опции для установки расписания доступны в [документации библиотеки `schedule`](https://schedule.readthedocs.io/en/stable/) или в файле `option_schedule.md` (есть в этом каталоге).

Примеры расписаний:

*   `schedule.every(10).minutes.do(my_job)`: каждый 10 минут
*   `schedule.every().hour.do(my_job)`: каждый час
*   `schedule.every().monday.do(my_job)`: каждый понедельник
*   `schedule.every().wednesday.at("13:15").do(my_job)`: каждую среду в 13:15

## Дополнительно

### Логи и Вывод
*   Консоль: Если `stream_on` в `config.json` установлен в 1 (или не указан), важные сообщения будут выводиться в консоль.
*   Файл логов: Все действия скрипта, включая ошибки, прогресс и обнаруженные изменения, записываются в файл, указанный в `log_file` в `config.json`.
*   Файлы состояний (`state_*.json`): Содержат список всех отслеживаемых файлов с их датами последнего изменения.
*   Файлы удаленных файлов (`removed_files_*.txt`): Содержат список путей к файлам, которые были обнаружены как удаленные с момента предыдущего запуска.
*   Файлы измененных файлов (`changed_files_*.txt`): Содержат список путей к файлам, которые были обнаружены как измененные (то есть их дата модификации отличается), с указанием старой и новой даты.
*   Для вывода более подробной информации(о пропущенных файлов и каталогах) следует найти в файле `file_monitor.py` строку `level_log = logging.INFO` и заменить её на `level_log = logging.DEBUG`.


### Ротация Файлов
*   Скрипт автоматически ротирует файлы логов, состояний, списков удаленных и измененных файлов. 
    При каждом успешном цикле мониторинга, старые версии этих файлов переименовываются с добавлением суффикса _XX (где XX - порядковый номер, начиная с 01), 
    а самые старые версии удаляются, если их количество превышает заданные лимиты (`rotare_state_files`, `rotare_remove_files`, `rotare_change_files`, `rotare_log_files`).
    Пример ротации `monitor_files.log` при `rotare_log_files`: 10:
    При следующем запуске `monitor_files.log` становится `monitor_files_01.log`.
    `monitor_files_01.log` (старый) становится `monitor_files_02.log`.
    …
    `monitor_files_09.log` (старый) становится `monitor_files_10.log`.
    `monitor_files_10.log` (самый старый) удаляется.
    Создается новый пустой `monitor_files.log`.

*   **Файлы состояний**: Скрипт управляет ротацией файлов состояний. При каждом запуске, текущий `state.json` становится предыдущим (`state_01.json`), 
    а старые файлы сдвигаются. Количество хранимых файлов определяется `rotare_state_files`.
*   **Списки удаленных файлов**: Аналогично, `removed_files_file` также подвергается ротации согласно `rotare_remove_files`.

### Игнорирование Каталогов/Файлов

*   Скрипт автоматически пропускает каталоги и файлы, указанные в `ignore_dirs` и `ignore_patterns` вашего `config.json`. 
    Это полезно для исключения временных файлов, системных каталогов (`.venv`) или других ненужных элементов из мониторинга.

#### Поддерживаемые Операционные Системы
    Скрипт разработан с использованием стандартных библиотек Python и должен корректно работать на любой операционной системе, где установлен Python (Windows, Linux, macOS).
