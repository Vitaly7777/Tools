# Описание #
*   `excel_processor_app.py` — это графическое пользовательское приложение на Python, предназначенное для объединения данных из нескольких файлов Excel в один и сохранения их в новый файл. 
    Приложение предоставляет гибкие настройки для определения того, откуда начинать извлечение данных, сколько столбцов извлекать, а также позволяет применять фильтрацию и включать заголовки.

*   С помощью этого инструмента Вы можете легко автоматизировать рутинные задачи по сбору данных из однотипных Excel-отчетов.

## Основные возможности
*   **Выбор нескольких исходных файлов Excel: Объединяйте данные из любого количества файлов .xlsx или .xlsm.**
*   **Гибкое определение начала данных и тапа данных:**

    *  Поиском по текстовому маркеру: Укажите текст, который находится в первой ячейке нужной вам таблицы данных. Приложение найдет эту ячейку и начнет извлечение с нее.*
    *  По явным координатам ячейки: Если позиция данных фиксирована, вы можете указать конкретную строку и столбец для начала извлечения.
    *  Визуальный предпросмотр и выбор: Специальный диалог предпросмотра позволяет видеть содержимое листа и интерактивно выбирать ячейку маркера/начала данных.
    *  Возможность преобразовать указанные столбцы в числовые типы данных(указанные к преобразованию столбцы помечаются *).

*    **Извлечение заданного количества столбцов: Укажите, сколько столбцов данных нужно извлечь, начиная с маркера.**
*   **Вставка заголовка: Возможность включить строку заголовка из исходного файла в результирующий файл.**

*   **Фильтрация данных:**

    *  Без фильтрации
    *  Удалить строки с пустым ПЕРВЫМ столбцом, (столбец, с которого начинается извлечение)
    *  Удалить ПОЛНОСТЬЮ пустые строки, (удаляет строки, в которых все извлеченные столбцы пусты)

*    **Конфигурация выходного файла: Выберите директорию и имя для объединенного файла.**
*    **Автофильтр в итоговом файле: В объединенный файл автоматически добавляется автофильтр для удобства анализа.**
*    **Система пресетов: Сохраняйте и загружайте часто используемые конфигурации (наборы настроек) для        различных типов файлов/задач. Последний использованный пресет загружается автоматически.**
*    **Подробное логирование: Отслеживайте ход выполнения и возникающие сообщения прямо в приложении.**
## Требования 
    Python 3.x
    PySide6
    Pandas
    Openpyxl
    Numpy

 ##   Установка

    1.  Создайте виртуальное окружение (рекомендуется):
        python -m venv venv

    2.  Активируйте виртуальное окружение: 
        *   На Windows:
            .\.venv\Scripts\activate
        *   На macOS/Linux:
            source .venv/bin/activate

    3.  Установите необходимые библиотеки:
        pip install -r requirements.txt
        или
        pip install PySide6 pandas openpyxl numpy

##  Запуск приложения
    После установки зависимостей вы можете запустить приложение, выполнив главный скрипт:
        python excel_processor_app.py

##  Использование
    При первом запуске приложение загрузит <Новый пресет> или последний использованный.

### 1. Управление пресетами настроек 

*   `Выбор пресета`: Используйте выпадающий список для загрузки сохраненных настроек или выбора <Новый пресет> для сброса всех параметров.
*   `Сохранить пресет`: Сохраняет текущие настройки в новый пресет. Если выбран существующий пресет и настройки изменились, будет предложено перезаписать его или сохранить как новый.
*   `Удалить пресет`: Удаляет выбранный пресет.

### 2. Выберите исходные файлы 
*   `Выбрать/Добавить файлы`: Откроет диалог выбора файлов. Вы можете выбрать один или несколько Excel-файлов (.xlsx, .xlsm). Если уже выбраны файлы, приложение спросит, хотите ли вы добавить к ним новые или заменить.
*   `Очистить список файлов`: Удаляет все выбранные файлы из списка.

### 3. Настройка параметров извлечения данных
*   `Лист для обработки`: Выпадающий список, который автоматически заполняется названиями листов из первого выбранного исходного файла. Выберите лист, с которого нужно извлекать данные.
*   `Использовать координаты ячейки`: Установите этот флажок, чтобы определять начало данных по номеру строки и столбца (1-based). Если флажок снят, данные будут искаться по текстовому маркеру.
*   `Предпросмотр, выбор маркера/координат и столбцов с числовыми данными`: Открывает диалог предпросмотра первого Excel-файла.
    * Кликните на ячейку на предпросмотре. Текст из нее будет использоваться как маркер (если флажок "Использовать координаты", снят).
    * Координаты выбранной ячейки (строка, столбец) будут использоваться как начало данных (если флажок "Использовать координаты", установлен).
    * Кликните по заголовкам('A', 'B' и т.д.) столбцов, содержащим числовые данные(выбранные к преобразованию столбцы помечаются '*').
*   `Количество столбцов для извлечения`: Укажите число столбцов, которые необходимо извлечь, начиная с ячейки маркера.
*   `Вставлять строку-заголовок`: Если активно, первая строка, извлеченная из первого файла (с учетом маркера), будет использоваться как заголовок для объединенного DataFrame.

### 4. Фильтрация данных

    Выберите один из вариантов фильтрации:

*    `Без фильтрации`: Все извлеченные строки будут сохранены.
*    `Удалить строки с пустым ПЕРВЫМ столбцом`: Удаляет строки, у которых первый извлеченный столбец пуст.
*    `Удалить ПОЛНОСТЬЮ пустые строки`: Удаляет строки, в которых все извлеченные столбцы данных (кроме 'Source_File') не содержат значений.*

### 5. Сохранение результата
*   `Выбрать файл для сохранения`: Откроет диалог, где вы можете указать путь и имя для итогового объединенного файла Excel.

### 6. Запуск и результат

*   Запустить обработку: Начинает процесс объединения и сохранения файлов согласно заданным настройкам.
*   Открыть созданный файл: Активируется после успешного сохранения. Позволяет открыть результирующий файл в приложении по умолчанию.
*   Лог: Отображает подробную информацию о процессе обработки, предупреждениях и ошибках.

####   Структура проекта:

`excel_processor_app.py`: Основной файл приложения с GUI и логикой обработки Excel-файлов.

`global_config.json`: Файл для хранения пресетов настроек и последнего использованного пресета.

`requirements.txt`: Файл с необходимыми библиотеками.
